# Task 6: Create a Strong Password and Evaluate Its Strength

## 📌 Project Overview

This project was completed as part of the **Cyber Security Internship** by **Elevate Labs**.

The objective of this task is to understand the characteristics of strong passwords by creating passwords with different complexity levels and evaluating them using an online password strength checker. The results are analyzed to understand how password length, character variety, and uniqueness contribute to password security.

---

## 🎯 Objective

- Create passwords with varying levels of complexity.
- Test each password using an online password strength checker.
- Analyze password strength results.
- Learn password security best practices.
- Understand common password attacks.

## 🛡️ Password Best Practices

- Use at least 12–16 characters.
- Combine uppercase and lowercase letters.
- Include numbers and special characters.
- Avoid dictionary words and personal information.
- Use a unique password for every account.
- Enable Multi-Factor Authentication (MFA).
- Store passwords securely using a password manager.

---

## ⚠️ Common Password Attacks

### Brute Force Attack

A brute force attack systematically tries every possible password combination until the correct password is found. Longer and more complex passwords significantly increase the time required to crack them.

### Dictionary Attack

A dictionary attack attempts passwords from a predefined list of commonly used words and leaked passwords. Passwords based on common words or predictable patterns are highly vulnerable to this attack.

---

## ✅ Conclusion

This project demonstrates how password complexity directly affects security. Weak passwords can be compromised quickly, whereas long, unique, and complex passwords provide significantly better protection against common password attacks. Following password security best practices helps safeguard personal and organizational data.
